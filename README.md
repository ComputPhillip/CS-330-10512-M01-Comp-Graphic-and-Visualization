A showcase of computational graphics and visualization

## Reflection

### How do I approach designing software?
I approach software design by first understanding the problem and identifying the core requirements. From there, I break the system into smaller components and define how they should interact. I focus on structure early, making sure responsibilities are clearly separated so the design stays organized and scalable. Instead of jumping straight into coding, I outline the architecture and think about how future changes might affect the system.

### What new design skills has this project helped me develop?
This project strengthened my ability to design systems visually and structurally at the same time. Working on a 3D scene required me to think about object relationships, transformations, lighting, and rendering pipelines as connected parts of a larger system. I improved at modular design by separating mesh creation, scene management, and rendering logic, which made the project easier to manage and extend.

### What design process did I follow?
My process followed a layered approach: plan → prototype → refine. I began by outlining the scene layout and technical requirements. Then I built a minimal working version to ensure the rendering pipeline functioned correctly. After that, I incrementally added complexity, such as additional objects, textures, and lighting adjustments, testing each addition before moving forward.

### How can this design approach be applied in future work?
The layered and modular approach can be applied to any future software project. Starting with a minimal viable version reduces complexity and prevents major integration issues later. Designing components with clear responsibilities also makes debugging and feature expansion more efficient. This process supports scalability and long-term maintainability.

### How do I approach developing programs?
I approach development iteratively. I implement one feature at a time, test it, and confirm that it works before building on top of it. This prevents large-scale errors and keeps the codebase stable. I also focus on readability and organization so that future modifications are easier to implement.

### What new development strategies did I use while working on the 3D scene?
While developing the 3D scene, I relied heavily on incremental rendering tests. I verified transformations, camera positioning, and lighting using single objects before integrating the full environment. I also used reusable helper functions and structured scene management to avoid repeating logic. Visual debugging became an important strategy, since graphical errors often reveal issues in calculations or object state.

### How did iteration factor into development?
Iteration was central to the workflow. Small adjustments to transformations, lighting intensity, or camera angles significantly impacted the final output. Repeated testing and refinement allowed the scene to gradually improve in realism and accuracy. This iterative cycle ensured steady progress and minimized major setbacks.

### How did my development approach evolve throughout the milestones?
Early in the project, my focus was primarily on functionality—ensuring objects rendered correctly. As the milestones progressed, I shifted toward writing cleaner, more organized code and planning features before implementing them. I became more intentional with structure and documentation, which made the final project more stable and cohesive.
